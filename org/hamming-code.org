#+Title: hammming code

为了检测数据传输中的错误，可以引入冗余编码，最常见的有以下几种:
1. 奇偶校验: 引入一个多余的位做奇偶校验，保证所有bit异或之后等于0，只能检错，不能纠错。
2. two-out-of-five: 这是一种比较奇特的编码方式，用5个bit表示10个数字，保证5个bit中有两个1，三个0。
3. 重复编码: 把一个bit重复k次，是最直接的冗余编码方法。
4. hamming码: 对2^n bit数据(包括校验位)使用n bit的冗余位做校验，可以检查并修复1个bit错误。
hamming码的想法是： 2^n bit的数据，单个bit错误的情况共有2^n 种可能，这么多种情况只需要用n bit就可以表示出来。
如果每个bit错误都会唯一对应到n个校验位的2^n 种状态，那么根据校验结果就可以知道出错的位了。

事实上hamming码最后的方案可以做到把n位校验位的校验结果拿出来组成一个2进制数，这个2进制数就直接指明了出错的位的位置。
而要做到这一点就意味着，如果第m位出错了，并且m的二进制表示中第i位为1，那么第i位校验位就要能检查出来. 
换言之第i位校验位负责校验二进制表示中第i位为1的那些位的正确性。因为只要求能纠正1位错误，所以使用奇偶校验即可。
